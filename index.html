SDF — Smart Document Format

An open format that embeds structured JSON alongside a visual PDF in a single self-validating file. General purpose — invoices, government forms, contracts, nominations, and more. The spec is open under CC-BY 4.0; the reference implementation is source-available under BUSL-1.1.

etapsky/sdf Spec · CC-BY 4.0 Code · BUSL-1.1

sdf-kit

TypeScript SDK for producing and consuming SDF files. Works in Node.js, browser, and Electron. Includes schema validation, producer pipeline, and reader utilities.

About
Built step by step
01

Open spec, source-available code

The SDF format specification is open under CC-BY 4.0 — any developer or organization can implement it freely. The reference tools are source-available under BUSL-1.1, transparent and inspectable.

02

Developer-first

Every product starts with the developer experience. Simple APIs, clear documentation, no hidden complexity. If it's hard to integrate, it's not done yet.

03

Interoperability by design

SDF is an open standard, not a proprietary format. Any system can implement SDF support — the spec is public, the schema travels with the document, and there is no central dependency.

04

Long-term thinking

We ship working software in phases and iterate with real feedback. No premature abstractions, no feature bloat — solid, dependable tools that hold up over time.
